
The purpose of the assignment:
Learn to use the exception mechanism.

What should be done:
Perform the task in the project
12_ExceptionsDebuggingAndTesting / homework_12.1 / ConsoleCustomerList

Register in the ConsoleCustomerList project in the CustomerStorage class all possible options for protecting against incorrect data 
and premature completion of the addCustomer () add method.
The program should throw exceptions (Exception) if the following is incorrect:
the number of components in the transmitted data string,
phone number format,
email format.

Criteria for evaluation:
"Passed" - tests are successfully performed.
"For revision" - the task has not been completed, the tests are completed with an error.

Additional task:
The programmer has not written tests for all cases of data entry. 
Find the case that throws an exception when you type one of the list commands.
Write a try ... catch block for this exception.

Цель задания:
Научиться использовать механизм исключений.

Что нужно сделать:
Задание выполняйте в проекте 
12_ExceptionsDebuggingAndTesting/homework_12.1/ConsoleCustomerList

Пропишите в проекте ConsoleCustomerList в классе CustomerStorage все возможные варианты защиты от некорректных данных 
и преждевременного завершения метода добавления addCustomer().
Программа должна выбрасывать исключения (Exception) при неверном:
количестве компонентов в переданной строке с данными,
формате номера телефона,
формате email.

Критерии оценки:
«Зачёт» — тесты успешно выполняются.
«На доработку» — задание не выполнено, тесты завершаются с ошибкой.

Дополнительное задание:
Не на все случаи ввода данных программист написал тесты. 
Найдите случай, который выбрасывает исключение при вводе одной из команд для работы со списком. 
Напишите try...catch блок для этого исключения.
